

## System Alshbh — خطة البناء (المرحلة الأولى: الأساسيات)

### الرؤية
مركز قيادة احترافي بتصميم Dark Mode Neon مع واجهة عربية كاملة (RTL)، مُحسّن للموبايل أولاً مع دعم Capacitor للتطبيق الأصلي.

---

### 🔧 ما سيتم بناؤه في هذه المرحلة

#### 1. التصميم الأساسي والهيكل
- **نظام ألوان Dark Neon** — خلفية سوداء عميقة مع حدود مضيئة وألوان Cyan/Violet/Green
- **خط IBM Plex Sans Arabic** مع دعم RTL كامل
- **Bottom Navigation** للموبايل (Dashboard, المشاريع, زر + المركزي, المالية, المزيد)
- **Sidebar** لشاشات الكمبيوتر
- **أنيميشن سلسة** باستخدام Framer Motion

#### 2. نظام تسجيل الدخول والأمان
- تسجيل دخول بالبريد وكلمة المرور عبر Supabase Auth
- حماية جميع الصفحات (Protected Routes)
- **قفل التطبيق** برمز PIN عند الخمول
- **سجل تسجيل الدخول** — تسجيل IP، الجهاز، المتصفح، الموقع، الحالة
- عرض وحذف السجلات من لوحة الإعدادات

#### 3. لوحة التحكم (Dashboard)
- إحصائيات سريعة: عدد المشاريع، الأرباح اليومية/الشهرية، المشاريع النشطة
- بطاقات مضيئة بتصميم Obsidian Glass
- تنبيهات الخدمات المنتهية والمهام القادمة

#### 4. إدارة المشاريع
- إضافة/تعديل/حذف مشروع مع كل التفاصيل (الاسم، الحالة، السعر، المدفوع/المتبقي، التواريخ، التصنيف)
- ربط المشروع بعميل
- ملاحظات لكل مشروع
- بطاقات مشاريع بتأثير Neon Pulse للمشاريع النشطة

#### 5. إدارة العملاء
- إضافة/تعديل/حذف عميل (الاسم، الهاتف، ملاحظات)
- زر واتساب مباشر لكل عميل
- عرض مشاريع ومدفوعات كل عميل

#### 6. قاعدة البيانات (Supabase)
- جداول: profiles, projects, clients, transactions, security_logs
- سياسات RLS للمستخدم الواحد
- إعداد Supabase Storage للملفات (مرحلة لاحقة)

#### 7. إعداد Capacitor
- تهيئة Capacitor للتطبيق الأصلي (iOS + Android)
- إعدادات Hot Reload من sandbox

---

### 📋 المراحل القادمة (بعد الأساسيات)
- النظام المالي الكامل (دخل/مصروف/تقارير)
- إدارة الخدمات (Supabase/Domains/Hosting + تنبيهات انتهاء)
- نظام الملفات (Supabase Storage)
- المهام والإشعارات
- الفواتير PDF + مشاركة واتساب
- البحث والفلاتر الشاملة
- التصدير (PDF/Excel)
- الرسوم البيانية والتحليلات الذكية
